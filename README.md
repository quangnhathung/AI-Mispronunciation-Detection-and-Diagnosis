# MDD_AI: Mispronunciation Detection and Diagnosis

The MDD_AI project is an Artificial Intelligence system that supports Mispronunciation Detection and Diagnosis (MDD) for second-language (L2) learners. The system is designed to evaluate each phoneme based on audio input and a canonical phoneme sequence, precisely identifying where the user mispronounced.

---

## 1. Model Architecture Overview

The model in this project (version V2/V3) follows a **Phoneme-Audio Alignment & Scoring** architecture, divided into 3 main modules:

1. **The Ears – Audio Feature Extractor (Wav2Vec 2.0)**:
   - Uses the pre-trained `facebook/wav2vec2-base-960h` model.
   - **Optimization**: Freezes the feature extractor and the first 10 transformer layers to preserve global audio perception and minimize overfitting on small L2 datasets. The output is a sequence of audio embedding vectors (Latent Audio).

2. **The Judge – Phoneme Encoder with Bi-GRU**:
   - Transforms canonical phoneme IDs into static embedding vectors.
   - Passes them through a **Bi-directional GRU** (Bidirectional Gated Recurrent Unit). This is a key upgrade that gives the model awareness of the "position" and "surrounding context" (co-articulation) of each phoneme before comparing it against the audio.

3. **Alignment and Scoring (Cross-Attention & Scoring Head)**:
   - **Cross-Attention**: Uses phoneme vectors (Query) to align with audio vectors (Key & Value), locating the audio segment that corresponds to each phoneme.
   - **Scoring Head**: An MLP network (Linear → ReLU → Dropout → Linear) takes the aligned feature vector and predicts the probability of correct/incorrect pronunciation (Binary Classification).

## 2. Comprehensive Details on the L2-ARCTIC Dataset

This project uses **L2-ARCTIC**, a benchmark corpus in the field of English pronunciation research for L2 learners.

### Overview (Basic)

L2-ARCTIC contains recordings from non-native English speakers from a variety of language backgrounds (Indian, Korean, Arabic, Spanish, Mandarin, Vietnamese, etc.). The goal of the dataset is to provide a realistic measure of how L2 speakers make errors when reading English sentences.

### Data Structure and Labels (Advanced)

- **Data format**: Each audio recording (`.wav`) is accompanied by a `.TextGrid` annotation file (generated by the Montreal Forced Aligner and subsequently refined manually by linguistics experts).
- **Error classification**: L2-ARCTIC annotates errors at the phoneme level, covering 3 main error types:
  - **Substitution**: Mispronouncing one phoneme as another (e.g., `d,t,s` — should be pronounced as `D`, but produced as `T` or `S`).
  - **Deletion**: Dropping a phoneme entirely (e.g., `k,*` — the phoneme `K` is present in the word but the speaker skips it).
  - **Addition**: Inserting phonemes not present in the dictionary (commonly seen when speakers incorrectly link sounds or add extra syllables).
- **How the project processes TextGrid files**: The `L2ArcticDataset` class scans through TextGrid files and extracts exactly the "phones" tier (ignoring "words" or "ipa" tiers). Labels are binarized:
  - Clean phonemes (e.g., `K`, `AA`) → Label `1.0` (Correct).
  - Phonemes flagged with an error comma (e.g., `sil,K,a`) → Label `0.0` (Incorrect).
- **Exception handling (Robustness)**: The dataset class is built with a try-except handler to automatically catch corrupted TextGrid files, returning a safe silence sequence (`SIL`) instead of crashing the entire training process.

## 3. Comprehensive Environment Setup Guide

To ensure audio libraries (such as `soundfile`) work smoothly and avoid version conflicts with PyTorch, use `conda` to isolate the environment.

### System Requirements:

- Operating System: Linux / Windows / macOS
- GPU: An NVIDIA GPU (RAM > 8GB) is recommended for training.

### Step 1: Create a Virtual Environment with Conda

Open a terminal and run the following command (the `-y` flag auto-accepts all installation prompts):

```bash
conda create -n mdd_ai python=3.11 -y
conda activate mdd_ai
```

### Step 2: Install PyTorch (Latest Version with CUDA Support)

Note: The command below installs the version with CUDA 12.1 support. If you don't have a GPU or are on macOS, omit the index URL.

```bash
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
```

### Step 3: Install Required Project Libraries

These libraries handle TextGrid processing, audio loading, and Transformers:

```bash
pip install transformers soundfile tgt tqdm
```

## 4. Starting the Training Process

- The training pipeline has a built-in **Penalty Weight** mechanism in the loss function to address class imbalance (correct pronunciations always significantly outnumber incorrect ones).
  - Actual errors are penalized with a weight 4× higher (`ERROR_WEIGHT = 4.0`).
  - The model saves the best checkpoint based on the **F1-Score of the Error detection class** (catching real mistakes) rather than the overall loss.
- To start training:

```bash
from main import train_model_v3

# Launch the training process for 25 epochs
train_model_v3(epochs=25)
```

**The best checkpoint will be saved at: `./checkpoints/best_mdd_model_v3.pt`**

```bash
./checkpoints/best_mdd_model_v3.pt
```
